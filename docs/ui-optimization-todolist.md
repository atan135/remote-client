# UI 界面优化 TodoList

更新时间：2026-05-09

- [x] 终端页：目标设备栏、一次性命令、交互式会话、预设命令对齐。
- [x] 对话页：当前设备、输入区、历史结果、执行状态层级重整，高级选项折叠。

## P0：优先处理

- [ ] 任务页 / 命令时间线
  - 目标：让筛选、命令状态、结果摘要更适合高频排查。
  - 建议：顶部做紧凑筛选栏，卡片首屏只展示设备、命令、状态、时间和关键输出。
  - 建议：长输出、stdout、stderr、错误详情统一进入详情弹窗。
  - 验收：手机端打开后能先看到筛选和最近任务，不被大块说明或过长结果挤掉。

- [x] 命令详情弹窗统一
  - 目标：对话页、任务页里查看命令结果时体验一致；终端交互会话界面保持不变。
  - 建议：统一为“元信息条 + 输出区域 + 原始命令”的结构。
  - 建议：stdout / stderr / error 分区更清晰，长输出可滚动，移动端全屏弹窗。
  - 验收：失败任务能一眼看到失败原因，成功任务能快速复制关键输出。

- [x] 全局空状态 / 加载态 / 错误态
  - 目标：减少“空白卡片”和零散提示文案。
  - 建议：设备为空、会话为空、命令为空、授权为空、用户为空时使用统一空状态样式。
  - 建议：加载中使用轻量骨架或固定高度状态区，避免布局跳动。
  - 验收：每个主页面在无数据、加载中、接口失败时都有明确状态。

## P1：中优先级

- [ ] 我的页里的授权管理
  - 目标：降低 RSA 公钥绑定的表单压迫感。
  - 建议：列表只展示 agentId、备注、指纹、更新时间和操作。
  - 建议：新增 / 编辑放到弹窗或独立面板，PEM 文本域默认只在编辑时出现。
  - 验收：普通用户能快速判断哪些设备已绑定，管理员不会被长文本干扰。

- [ ] 我的页里的用户管理 / 设备审批
  - 目标：管理型信息更易扫读。
  - 建议：桌面端可保留表格密度，手机端改为列表卡片。
  - 建议：启停、重置密码、审批等高风险操作统一确认弹窗文案。
  - 验收：手机端不出现横向溢出，危险操作不与普通操作混在一起。

- [ ] 远程文件预览弹窗
  - 目标：更像文件查看器，而不是普通内容弹窗。
  - 建议：顶部固定文件名、路径、大小、编码等元信息。
  - 建议：正文区域独立滚动，Markdown、纯文本、空文件状态一致。
  - 建议：增加复制路径、复制内容等常用动作。
  - 验收：大文件、长路径、Markdown 文件在桌面和手机端都能稳定查看。

- [ ] 对话页与终端页的细节复核
  - 目标：针对真实截图再做一轮对齐与首屏密度校准。
  - 建议：重点看手机端首屏、按钮宽度、状态标签换行、长设备名截断。
  - 验收：常用入口不需要滚动太多即可完成主要操作。

## P2：后续打磨

- [ ] TopBar / BottomTabBar 移动端导航
  - 目标：让手机端顶部标题、底部导航与页面主内容更协调。
  - 建议：检查安全区、激活态、长标签、顶部状态提示挤压问题。

- [ ] 全局按钮与图标体系
  - 目标：减少文字按钮堆叠，提高工具型界面的识别效率。
  - 建议：常用操作使用图标 + tooltip，高风险操作保持文字明确。

- [ ] 全局间距与卡片规范
  - 目标：统一 `surface-card`、表单区、详情区、列表项的 padding、圆角和边框。
  - 建议：清理历史遗留样式，减少页面之间同类组件的视觉差异。

- [ ] 可访问性与键盘操作
  - 目标：提升表单、弹窗、命令输入的键盘可用性。
  - 建议：检查 focus 状态、Enter / Ctrl+Enter、Esc 关闭弹窗、Tab 顺序。

- [ ] 视觉回归检查清单
  - 目标：每轮 UI 改动后能快速检查桌面和手机端。
  - 建议：固定检查宽度：1440、1024、460、390、360。
  - 建议：覆盖页面：首页、终端、对话、任务、我的、登录、注册。
